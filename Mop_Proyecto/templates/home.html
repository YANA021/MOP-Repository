{% trans "Programación Lineal y Método Gráfico" %}

{% blocktrans trimmed %} La programación lineal (PL) surge durante la Segunda Guerra Mundial como una herramienta para resolver problemas logísticos como el transporte y la dieta de los soldados. Su desarrollo se atribuye a George Dantzig, quien propuso el Método Simplex, el cual más adelante fue adoptado por las empresas para tomar decisiones científicas. {% endblocktrans %}

{% blocktrans trimmed %} Desde los años 50, la PL se integró al ámbito empresarial y, gracias a los avances tecnológicos y la globalización, se ha vuelto esencial en la toma de decisiones en sistemas productivos complejos. Su utilidad radica en que permite planificar actividades para maximizar beneficios o minimizar costos respetando las restricciones existentes. {% endblocktrans %}

{% blocktrans trimmed %} La PL se aplica a problemas que incluyen variables no negativas, sujetas a restricciones lineales, y busca optimizar una función objetivo. Para que un problema sea considerado de programación lineal deben cumplirse condiciones como: recursos limitados, objetivo definido, linealidad, homogeneidad y divisibilidad. {% endblocktrans %}

{% trans "Un modelo de PL incluye tres componentes fundamentales:" %}

  • {% blocktrans trimmed %} Función objetivo: maximizar o minimizar valores como ganancias o costos. {% endblocktrans %}
  • {% blocktrans trimmed %} Actividades posibles: opciones disponibles dentro del sistema (producción, ventas, etc.). {% endblocktrans %}
  • {% blocktrans trimmed %} Restricciones: limitaciones físicas, de mercado o políticas que afectan el modelo. {% endblocktrans %}

{% blocktrans trimmed %} En cuanto al método gráfico, se utiliza para resolver problemas de PL con dos variables de decisión (x₁ y x₂). Este método representa las restricciones en un plano cartesiano, permitiendo visualizar la región factible y encontrar la solución óptima. {% endblocktrans %}
